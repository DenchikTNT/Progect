Це технічне резюме (Technical Summary) проєкту "NEON BATTLE PRO" — динамічного мережевого 2D-шутера, розробленого на мові програмування Python.

🛠 Технологічний стек
Мова програмування: Python 3.x

Графічний інтерфейс: customtkinter (сучасний UI) та tkinter.Canvas (ігровий рушій).

Мережева взаємодія: Бібліотека socket (протокол TCP) та багатопотоковість threading.

Робота з медіа: pygame (звукові ефекти, музика) та Pillow (PIL) (рендеринг та масштабування зображень).

🏗 Архітектура системи
Проєкт розділений на дві незалежні частини за моделлю Client-Server:

1. Центральний сервер (server.py)
Функція: Виступає хабом для обміну даними між гравцями.

Логіка:

Слухає вхідні підключення на порту 6666.

Призначає унікальні ID (0 або 1) для ідентифікації гравців.

Використовує багатопотоковість для одночасного обслуговування обох клієнтів.

Транслює пакети даних (рух, постріли, влучання) від одного гравця іншому в реальному часі.

2. Клієнтська програма (game.py)
Ігровий цикл (Main Loop): Працює на частоті ~60 FPS через метод .after().

Мережевий потік: Окремий потік recv постійно очікує дані від сервера, щоб оновлювати позицію ворога без затримок в інтерфейсі.

Фізика та колізії: Реалізована перевірка перекриття об'єктів (find_overlapping) для детекції влучань у стіни або ворожий корабель.

🎮 Ігрові механіки
Мережевий рендеринг: Кожен клієнт обробляє власну фізику руху, але передає свої координати через сервер для візуалізації на екрані суперника.

Система стрільби: Кулі генеруються як незалежні об'єкти в окремих потоках, що дозволяє випускати кілька снарядів одночасно без блокування керування.

Адаптивний дизайн: Реалізовано динамічне масштабування фонового зображення та меж ігрового поля при зміні розміру вікна.

Аудіо-супровід: Розділені канали для фонової музики (циклічно) та звукових ефектів (паралельно).

📡 Протокол передачі даних
Дані передаються у вигляді коротких рядкових пакетів для мінімізації затримки (Ping):

m|x|y — оновлення позиції корабля.

s|x|y|dx|dy — створення нового снаряда з вектором напрямку.

hit| — сигнал про успішне влучання та оновлення рахунку.

📈 Перспективи розвитку
Впровадження протоколу UDP для зменшення мережевих затримок.

Додавання системи бонусів (Power-ups) та різних типів озброєння.

Створення системи лобі для підключення більше ніж двох гравців.
